- LDAP: Lightweight Directory Access Protocol. É um protocolo de aplicaçăo aberto, livre de fornecedor e padăo de 
industria para acessar e manter serviços de informaçăo de diretório distribuído sobre uma rede de Protocolo da Internet.
- Ataques MitM: Man in the Middle: É uma forma de ataque em que os dados trocados entre duas partes săo de alguma forma
interceptados, registrados e, possívelmente, alterados pelo atacante sem que as vítimas percebam.

- IAM: AWS Indentity and Access Managent é um serviço da Web que ajuda vocę a controlar o acesso aos recursos da AWS de forma segura.
Usamos o IAM para controlar quem é autenticado (fez login) e autorizado (tem permissőes) ao usar os recursos.

- Execuçăo arbitrária de código: É usado para descrever a capacidade de um invasor execute qualquer comando de escolha
do atacante em uma máquina de destino ou num processo de destino.É comumente usado em vulnerabilidade de execuçăo de 
código arbitrário para descrver um bug que dá ao invasor uma maneira de executar o código. A maioria dessas vulnerabilidades
permite a execuçăo de código de máquina e a maioria dos exploits injeta e executa shellcode para dar ao invasor
uma maneira fácil de executar comandos arbitrários manualmente.

-Cross-site Request Forgery (CSRF): Utiliza a confiança que um site tenha no navegador de um usário autenticado para 
ataque maliciosos. CSRF utiliza links ou scripts para enviar solicitaçőes HTTP involuntárias para um site de destino 
onde o usuário está autenticado.

- The Open Web Application Security Project ou OWASP é uma organizaçăo que trabalha com o objetivo de aprimorar a 
segurançă de software. Um dos trabalhos do grupo é atualizar um relatório com os top 10 maiores riscos de segurança
em uma aplicaçăo web.

A1 Injection: Falhas de injeçăo, tais como injeçőes de SQL,OS e LDPA ocorrem quando dados năo confiáveis săo 
enviados para um interpretador como parte de um comando ou consulta legítima. Os dados hostis do atacante podem
enganar o interpretador levando-o a executar comandos năo pretendidos ou a ceder a dados sem a devida autorizaçăo.
- Năo utilizar concatenaçăo de strings ao usar o banco de dados.
- Sanitization: Validar que o input do usuário năo é um script malicioso.
